# 定义和时间复杂度
数据结构的定义：抽象数据类型的物理表示，数据类型是包括数据集和对数据集的操作，抽象是不依赖具体实现，和机器、编程语言、数据存放的物理结构无关。

算法的定义：接受输入（或者无输入），产生输出的一段有限步骤的逻辑，会在有限时间内完成，每个步骤是明确的，没有二义性，是一种和编程语言无关的抽象逻辑表示。和程序的区别：程序可以无限执行，比如操作系统，算法必须在有限时间内完成。

算法的时间复杂度为T(n)，是完成算法所需的时间，n是算法处理的数据规模，和包括最坏时间复杂度和平均时间复杂度。

渐进上界定义如下，当n很大时，T(n)<=O(f(n))=c*f(n)，O(f(n))是T(n)的上界，f(n)是渐进上界，渐进上界不一定大于T(n)，用于表示时间复杂度，随着数据规模增加的增长趋势，

# 线性表
定义：同一类型元素的有序线性集合。包括数组、链表、栈、队列。

数组：每个元素在连续空间中存放。根据元素的位置访问的时间复杂度是O(1)，根据元素的值访问是O(n)，插入是O(n)，删除是O(n)。

链表：每个元素不在连续空间中存放，通过保存下一个元素的地址，来实现逻辑上的连续。根据元素的位置访问的时间复杂度是O(n)，根据元素的值访问是O(n)，插入是O(1)，删除是O(1)。

栈：只在一端进行访问、插入、删除，后进先出。

队列：一端进行插入，另一端进行访问、删除。

# 树
## 普通树
树的定义：树是n个节点的有限集合。有如下条件

+ n=0时，是空树。
+ n>0时，有1个根节点，其余集合可以分为m(m>=0)个子集合，子集合之间不相交，同时也满足树的定义，每个子集合是一棵子树。

树的相关术语：

+ 节点的度：节点的子树的个数。
+ 树的度：节点的度的最大值。
+ 叶节点：度为0的节点。
+ 父节点（Parent），子节点(Child)
+ 路径：从节点n1~nk是路径，其中ni是nj的父节点(1<=i<j<=k)，路径的长度是路径中边的个数。
+ 祖先节点（Ancestor）：由根节点到某一节点的路径中的所有节点，是这个节点的祖先节点。
+ 子孙节点（Descendent）：某个节点的子孙节点，是所有祖先节点是这个节点的节点。节点的祖先节点，只会形成1条路径，节点的子孙节点，可能形成多条路径。
+ 节点的层数：根节点的层数是1，如果父节点的层数是i，子节点的层数是i+1
+ 树的深度：节点层数的最大值

## 二叉树
### 定义和性质
普通树都可以使用二叉树来表示，如下，使用儿子-兄弟表示法，每个节点只保留第一个子节点和第一个兄弟节点。

二叉树(Binary Tree)的定义：是一棵树，并且每个节点最多有2个子树。

满二叉树（完美二叉树）：所有叶子节点所在的层数，等于树的深度的二叉树。

完全二叉树（Complete BinTree）：所有层数等于树的深度的叶子节点，在树的左侧的二叉树，如下。完全二叉树不一定是满二叉树，满二叉树一定是完全二叉树。完全二叉树可以使用顺序存储。

二叉树第i层节点的最大个数是2^(i-1)，因为第1层是1=2^0，第2层最多是2^1...。

深度为k的二叉树的节点数最多是2^k - 1。	

任意非空二叉树，如果叶子结点个数是n0，子节点数为2的节点数是n2，有n0=n2+1，直观理解：n=1的二叉树，n0=1，n2=0，后续每增加1个叶子节点，子节点数为2的节点数也会加1。

### 二叉树的遍历
先序、中序、后序遍历，先序：先访问根节点，再先序遍历左子树、右子树。中序：先中序遍历左子树，再访问根节点，中序遍历右子树。后序：后序遍历左子树、右子树，再访问根节点。

根据定义，得到递归实现，如下

```c
void PreOrderTraversalRecursion(BinTree binTree){
    if(binTree){
        Print(binTree);
        PreOrderTraversalRecursion(binTree->left);
        PreOrderTraversalRecursion(binTree->right);
    }
}
void MiddleOrderTraversalRecursion(BinTree binTree){
    if(binTree){
        MiddleOrderTraversalRecursion(binTree->left);
        Print(binTree);
        MiddleOrderTraversalRecursion(binTree->right);
    }
}
void PostOrderTraversalRecursion(BinTree binTree){
    if(binTree){
        PostOrderTraversalRecursion(binTree->left);
        PostOrderTraversalRecursion(binTree->right);
        Print(binTree);
    }
}
```

根据递归实现，可以得到，先序、中序、后序遍历时，经过节点的路径是一样的，只是访问节点的时机不同，如下，对于某个节点，先序遍历是第一次遇到它时访问，中序是第2次遇到它时访问，后序是第3次遇到它时访问，"第1次遇到"的含义是遍历左子树之前，"第2次遇到"的含义是遍历左子树之后，"第1次遇到"的含义是遍历右子树之后，因此先序、中序、后序非递归实现，只有访问节点的时机不同，其他逻辑可以完全相同，实现的时候，可以先按照递归的思路，实现经过节点的逻辑，再确定与先序、中序、后序遍历，相对应的访问时机。

![](/assets/images/1773547105707-9999e5d6-d427-4432-b5a6-40cc744cec6a.webp)

根据递归的思路，从根节点开始，始终递归遍历左子树，达到最左（左子树为空）的节点的时候，最左节点的左递归函数返回，然后遍历它的右子树，也就是把它的右孩子当成根节点，循环处理。

用栈来保存父节点-左子树的路径，如果某个节点的右孩子不为空，转向它的右孩子时，节点可以出栈也可以不出栈，如果不出栈，将来它的右子树遍历结束，右孩子出栈后，它也要出栈，因此需要额外空间，记录节点的右孩子是否已访问，如果转向它的右孩子之前，提前出栈，就不需要这个额外空间，先序和中序遍历不需要右孩子出栈后访问节点，因此可以提前出栈，后序遍历需要右孩子出栈后访问节点，因此不能提前出栈。

```c
void PreOrderTraversalNonRecursion(BinTree binTree){
    Stack st=AllocStack(100);
    BinTreeNode curr=binTree;
    while(!IsEmpty(st)||curr!=NULL){
        while(curr){//判断右孩子为空的逻辑在这里，这里既能判断右孩子，也能判断左孩子是否为空
            Push(st,curr);
            //根据递归的思想，来用栈实现非递归，调用节点左孩子的方法之前，访问节点
            Print(curr);
            curr=curr->left;
        }
        //这里栈一定不为空，因为如果curr!=NULL，会在前面的while中入栈
        //在调用节点的右孩子的方法前，节点提前出栈，因为右孩子的方法结束后，节点的方法也结束
        curr=Pop(st);
        //转到右孩子，将它当成根节点循环处理
        curr=curr->right;
    }

}
void MiddleOrderTraversalNonRecursion(BinTree binTree){
    Stack st=AllocStack(100);
    BinTreeNode curr=binTree;
    while(!IsEmpty(st)||curr!=NULL){
        while(curr){
            Push(st,curr);
            curr=curr->left;
        }
        curr=Pop(st);
        //curr是在访问右节点前，提前出栈的，对于中序遍历，正好可以访问
        Print(curr);
        curr=curr->right;
    }
}
void PostOrderTraversalNonRecursion(BinTree binTree){
    Stack st=AllocStack(100);
    BinTreeNode curr=binTree;
    BinTreeNode prev=NULL;
    while(!IsEmpty(st)||curr!=NULL){
        while(curr){
            Push(st,curr);
            curr=curr->left;
        }
        curr=Peek(st);//A
        //prev记录了最近一次访问的节点，curr->right==prev表示curr的右节点已经访问过了
        if(curr->right==NULL || curr->right==prev){
            Print(curr);
            Pop(st);
            prev=curr;
            curr=NULL;
        }else{
            curr=curr->right;
        }
    }
}
```

根据遍历的序列，反推得到二叉树，至少要有中序序列，以及先序和后序序列中的一个，比如中序序列和先序序列，先根据先序或后序序列得到根节点，然后根据中序序列得到左右子树的节点，根据左或右子树，从先序或后序得到左或右子树的根节点，直到确定每个节点。

层序遍历：先访问根节点，在层序遍历它的左子树和右子树。使用队列实现。

### 二叉搜索树
二叉搜索树（二叉排序数，BST树）是一棵二叉树，并且满足如下定义：

+ 根节点左子树每一个节点的值，小于根节点。
+ 根节点右子树每一个节点的值，大于根节点。
+ 根节点左、右子树，也是二叉搜索树

### 平衡二叉树
平衡二叉树（AVL树）是一棵二叉搜索树，并且任一节点的左右子树高度差的绝对值小于等于1，左右子树高度差的绝对值，叫做平衡因子（BF，Balanced Factor）。

AVL树的查找效率：最差情况的查找效率，实际上就是计算n个节点的AVL树的最大高度，也就是高度为h时，最小节点数nh和h的函数关系，nh=F(h+2)-1，Fh是斐波那契数列，证明如下

AVL树的实现：不需要父指针，只需要左右孩子的指针和节点的高度。因为插入和删除时，只需要判断一条路径上的节点是否不平衡，插入时，使用栈，保存根节点到插入节点的路径，节点插入后，从插入节点到根节点，回溯路径，更新每个节点的高度，判断左右子树的高度是否不平衡，如果不平衡，根据情况进行单旋或双旋，情况如下，假设节点A的左右子树高度不平衡

| | 条件 | 旋转方向 | 变换逻辑 |
| --- | --- | --- | --- |
| LL旋转 | A的左子树高度>A的右子树高度，并且左节点的左子树高度>左节点的右子树高度 | A右旋 | 父节点成为左节点的右节点，左节点成为子树根节点，左节点的右子树，接到父节点的左子树 |
| RR旋转 | A的左子树高度<A的右子树高度，并且右节点的左子树高度<右节点的右子树高度 | A左旋 | 父节点成为右节点的左节点，右节点成为子树根节点，右节点的左子树，接到父节点的右子树 |
| LR旋转 | A的左子树高度>A的右子树高度，并且左节点的左子树高度<左节点的右子树高度 | 先A的左节点RR旋转，然后A做LL旋转 |  |
| RL旋转 | A的左子树高度<A的右子树高度，并且右节点的左子树高度>右节点的右子树高度 | 先A的右节点LL旋转，然后A做RR旋转 |  |


旋转的作用：不改变BST的性质的前提下，改变左右子树的高度。旋转的代码如下

```cpp
void AVLTree::LLRotate(AVLTreeNode* node,AVLTreeNode** parent){
    AVLTreeNode* left=node->getLeft();
    node->setLeft(left->getRight());
    left->setRight(node);
    *parent=left;
}
void AVLTree::RRRotate(AVLTreeNode* node,AVLTreeNode** parent){
    AVLTreeNode* right=node->getRight();
    node->setRight(right->getLeft());
    right->setLeft(node);
    *parent=right;
}
void AVLTree::balance(AVLTreeNode* finder,AVLTreeNode** parent){
    //打印平衡前后的树
    if(AVLTree::isPrintBalance){
        std::cout<<"平衡前:"<<std::endl;
        prettyPrint();
    }
    if(finder->getBf()>0){ //左子树高度>右子树高度
        if(finder->getLeft()->getBf()>0){//finder->getLeft()的左子树高度>右子树高度
            //LL插入，需要做LL旋转
            LLRotate(finder,parent);
        }else{//finder->getLeft()->leftH < finder->getLeft()->rightH
            //LR插入，需要做LR旋转
            RRRotate(finder->getLeft(),&finder->left);
            LLRotate(finder,parent);
        }
    }else{//finder->leftH < finder->rightH
        if(finder->getRight()->getBf()>0){
            //RL插入
            LLRotate(finder->getRight(),&finder->right);
            RRRotate(finder,parent);
        }else{//finder->getRight()->leftH < finder->getRight()->rightH
            //RR插入
            RRRotate(finder,parent);
        }
    }
    if(AVLTree::isPrintBalance){
        std::cout<<"平衡后:"<<std::endl;
        prettyPrint();
    }
}
```

```cpp
AVLTreeNode* AVLTree::insert(int v){
    AVLTreeNode** currAddr=&this->head;
    AVLTreeNode* curr=*currAddr;
    Stack<AVLTreeNode**> st(100);
    while(1){
        if(curr==nullptr){
            *currAddr=new AVLTreeNode(v);
            break;
        }
        if(v==curr->data) return nullptr;
        if(v>curr->data){
            st.push(currAddr);
            currAddr=&curr->right;
            curr=*currAddr;
        }else if(v<curr->data){
            st.push(currAddr);//60,
            currAddr=&curr->left;
            curr=*currAddr;
        }
    }
    while(!st.isEmpty()){
        currAddr=st.pop();
        tryBalance(currAddr);
    }
    return nullptr;
}
void AVLTree::tryBalance(AVLTreeNode** currAddr){
    AVLTreeNode* curr=*currAddr;
    curr->computeH();
    int bf=std::abs(curr->getBf());
    if(bf>1){
        balance(curr,currAddr);
    }
}
```

```cpp
//删除的非递归实现
void AVLTree::remove(int deleteV){
    Stack<AVLTreeNode**> st(100);
    AVLTreeNode* curr=this->head;
    st.push(&this->head);
    while(curr!=nullptr && deleteV!=curr->data){
        if(deleteV>curr->data){
            curr=pushSt(st, &curr->right);
        }else if(deleteV<curr->data){
            curr=pushSt(st, &curr->left);
        }
    }
    //要删的不存在，删除失败
    if(curr==nullptr) return;
    if(curr->getLeft()!=nullptr&&curr->getRight()!=nullptr){
        AVLTreeNode* predecessor=pushSt(st, &curr->left);
        while(predecessor->getRight()){
            predecessor=pushSt(st, &predecessor->right);
        }
        curr->data=predecessor->data;
        curr=predecessor;
    }
    //curr一定有0个或1个孩子
    AVLTreeNode** parent=st.pop();
    //包含了0个或1个孩子的情况
    AVLTreeNode* child=nullptr;
    if(curr->getLeft()!=nullptr) child=curr->getLeft();
    else child=curr->getRight();
    *parent=child;
    delete curr;
    while(!st.isEmpty()){
        tryBalance(st.pop());
    }
}
```

### 红黑树
红黑树的定义：

1. 节点是红色或黑色
2. 根节点是黑色的
3. 空的孩子节点是黑色的nil节点
4. 红色节点的子节点一定是黑色
5. 从根节点到每个叶子节点的路径上，黑色节点个数相同

性质4表示，任何叶子节点的路径上，不能出现红红相邻，限制了每条路径上，红色节点数最多等于黑色节点数，性质5限制了每条路径，黑色节点数必须相同，由4和5可以得到，红黑树最长路径，小于等于最短路径的2倍，最长路径是红节点数达到最大，最短路径是红节点数等于0。

性质2不是必须的，根节点如果是红色，相当于每条路径减数1个黑色节点，也能实现最长路径小于等于最短路径的2倍，为了实现简洁和同一，将根定义为黑色，如果根是红色，插入时，判断父节点是红色后，还需要判断父节点是否是根节点。

性质3是必须的，如果不定义nil节点，会出现满足性质4和5，但明显不平衡的树，如下，并且不方便定义黑高。

```cpp
      10B
     /
    9R
   /
  8B
 /
7R
/
6B
```

如果补上nil，这棵树不满足性质5，如下

```cpp
      10B
     /   \
    9R   NIL
   /
  8B
 /
7R
/
6B
/ \
NIL NIL
```

红黑树的插入：插入的节点一定是红色，如果是黑色，会使得插入节点的路径的黑色节点数增加，不方便调整，插入节点的父节点如果是黑色，不需要调整，如果是红色，出现了红红相邻的情况，需要调整，根据插入节点的叔叔节点的颜色，分成4种情况，如下

| | 处理逻辑 |
| --- | --- |
| (1A,2A,3A,4A)叔叔节点是红色 | 将父节点和叔叔节点染黑，爷爷节点染红，然后将爷爷节点当成新插入的节点，递归处理 |
| 2B,叔叔节点是黑色，父节点是爷爷的左孩子，插入节点是父节点的左孩子 | 对爷爷节点做LL旋转，爷爷节点染红，父节点染黑 |
| 3B,叔叔节点是黑色，父节点是爷爷的左孩子，插入节点是父节点的右孩子 | 对父节点做RR旋转，转成2B |
| 4B,叔叔节点是黑色，父节点是爷爷的右孩子，插入节点是父节点的左孩子 | 对父节点做LL旋转，转成5B |
| 5B，叔叔节点是黑色，父节点是爷爷的右孩子，插入节点是父节点的右孩子 | 对爷爷节点做RR旋转，爷爷节点染红，父节点染黑 |

```cpp
    LL表示父节点是爷爷的左孩子，当前节点是父节点的左孩子。
    RR表示父节点是爷爷的右孩子，当前节点是父节点的右孩子。
    LR表示父节点是爷爷的左孩子，当前节点是父节点的右孩子。
    RL表示父节点是爷爷的右孩子，当前节点是父节点的左孩子。
    */
void handleBlackUncle(RbTreeNode* curr){
    RbTreeNode* parent=curr->getParent();
    RbTreeNode* grandPa=curr->getGrandPa();
    if(parent==grandPa->getLeft() && curr==parent->getLeft()){
        //LL, 变色+旋转
        parent->setBlack();
        grandPa->setRed();
        LLRotate(grandPa);
    }else if(parent==grandPa->getRight() && curr==parent->getRight()){
        //RR, 变色+旋转
        parent->setBlack();
        grandPa->setRed();
        RRRotate(grandPa);
    }else if(parent==grandPa->getLeft() && curr==parent->getRight()){
        //LR, 对父节点RR旋转，转成LL
        RRRotate(parent);
        //父节点和当前节点位置互换了，变色需要注意
        curr->setBlack();
        grandPa->setRed();
        LLRotate(grandPa);
    }else{ //if(parent==grandPa->getRight() && curr==parent->getLeft()){
        //RL，对父节点LL旋转，转成RR
        LLRotate(parent);
        //父节点和当前节点位置互换了，变色需要注意
        curr->setBlack();
        grandPa->setRed();
        RRRotate(grandPa);
    }
}
//插入节点的父节点是红色，非递归实现
void adaptForInsert(RbTreeNode* curr){
    while(true){
        //curr一定不是根，父节点一定存在
        RbTreeNode* parent=curr->getParent();
        //父节点是黑色，不破坏红黑树的性质
        if(RbTreeNode::isBlack(parent)) return;
        //父节点是红，一定不是根，爷爷节点一定存在
        RbTreeNode* grandPa=curr->getGrandPa();
        RbTreeNode* uncle=curr->getUncle();
        //空的孩子节点是黑色
        if(RbTreeNode::isBlack(uncle)){
            //叔叔是黑，经过变色+旋转后，爷爷是黑，不需要递归处理
            handleBlackUncle(curr);
            return;
        }
        if(RbTreeNode::isRed(uncle)){
            //叔叔节点是红色，只需要变色，将叔叔和父节点变黑，爷爷变红，然后把爷爷当成新插入的节点，递归处理
            uncle->setBlack();
            parent->setBlack();
            //爷爷是根节点，不需要变红，递归处理结束
            if(grandPa==this->root) return;
            grandPa->setRed();
            curr=grandPa;
        }
    }
}
void insert(int v){
    //根节点是黑色
    if(this->root==nullptr){
        RbTreeNode* toInsert=new RbTreeNode(v);
        toInsert->setBlack();
        this->root=toInsert;
        return;
    }
    RbTreeNode* parent=nullptr;
    RbTreeNode* res=search(v,&parent);
    //节点已存在，插入失败
    if(res!=nullptr) return;
    RbTreeNode* toInsert=new RbTreeNode(v);
    if(v>parent->getData()){
        parent->setRight(toInsert);
    }else{ //v<parent->getData()
        parent->setLeft(toInsert);
    }
    adaptForInsert(toInsert);
}
```

黑高的定义：节点到任意nil节点的路径中，黑色节点的数量，称为节点的黑高。

性质1：节点到任意nil节点的路径，黑色节点的数量一定相等。反证：如果不等，这2条路径，加上节点到根节点的路径，得到的2条路径，黑色节点不相等，违反定义。

性质2：兄弟节点的黑高一定相等。证明：如果父节点是黑色，兄弟节点的黑高，等于父节点的黑高-1，如果父节点是红色，兄弟节点的黑高，等于父节点的黑高，总之都相等。

删除：如果删除的是红色节点，可以直接删除，如果删除黑色节点，需要做调整。

双黑节点：删除节点后，删除位置的占位节点，它表示2个黑色节点，当删除黑的叶子节点时，为了保持黑节点个数不变，假设删除的位置有一个双黑节点，2种方法将双黑变成单黑：

1. 通过旋转，在双黑节点所在的路径，增加一个黑节点。
2. 将兄弟节点变红，黑向上传递，如果父节点是红，父节点染黑，结束，如果父节点是黑，则父节点变为双黑节点，递归处理。

根据兄弟节点和侄子节点的颜色，使用不同的处理逻辑，如下，p是父节点，x是双黑节点，w是兄弟节点，wL和wR是侄子节点。

| | 处理逻辑 |
| --- | --- |
| 1，兄弟节点是黑色，远侄子是红色 | 父节点做RR或LL旋转，交换父节点和兄弟节点的颜色，远侄子染黑 |
| 2，兄弟节点是黑色，远侄子是黑色，近侄子是红色 | 兄弟节点做RR或LL旋转，交换近侄子和兄弟节点的颜色，转成情况1 |
| 3、兄弟节点是黑色，远侄子是黑色，近侄子是黑色 | 兄弟节点染红（黑色上移），如果父节点是红色，父节点染红，结束，如果父节点是黑色，父节点变成双黑节点，递归处理 |
| 4、兄弟节点是红色，此时父节点和侄子一定是黑色 | 父节点做LL或RR旋转，交换父节点和兄弟节点的颜色，转成兄弟是黑色的情况（1,2,3） |

```cpp
void handleBlackSiblingRedFarNePhew(RbTreeNode* dBlack){
    RbTreeNode* parent=dBlack->getParent();
    RbTreeNode* sibling=dBlack->getSibling();
    RbTreeNode* farNePhew=dBlack->getFarNephew();
    //父节点旋转，交换兄弟和父节点的颜色，远侄子染黑
    if(dBlack==parent->getLeft()){
        //父节点左旋
        RRRotate(parent);
    }else{ //dBlack==parent->getRight()
        //父节点右旋
        LLRotate(parent);
    }
    sibling->setSameColor(parent);
    parent->setBlack();
    farNePhew->setBlack();
}
/*
处理兄弟节点是黑的情况
return: null表示不需要递归处理，否则表示需要把返回的节点，当成新的双黑节点处理
*/
RbTreeNode* handleBlackSibling(RbTreeNode* dBlack){
    RbTreeNode* parent=dBlack->getParent();
    RbTreeNode* sibling=dBlack->getSibling();
    RbTreeNode* farNePhew=dBlack->getFarNephew();
    RbTreeNode* neiboNephew=dBlack->getNeiboNephew();
    if(RbTreeNode::isRed(farNePhew)){
    //兄弟节点是黑，远侄子是红，可以通过旋转+变色来处理，不需要转成其他情况
        handleBlackSiblingRedFarNePhew(dBlack);
        return nullptr;
    }else if(RbTreeNode::isBlack(farNePhew)&&RbTreeNode::isRed(neiboNephew)){
        //兄弟节点是黑，远侄子是黑,近侄子是红，旋转兄弟节点+变色，转成情况1
        if(dBlack==parent->getLeft()){
            LLRotate(sibling);
        }else{
            RRRotate(sibling);
        }
        //交换近侄子和兄弟的颜色
        sibling->setRed();
        neiboNephew->setBlack();
        handleBlackSiblingRedFarNePhew(dBlack);
        return nullptr;
    }else{// isBlack(farNePhew)&&isBlack(neiboNephew)
        //兄弟节点染红，此时当前节点和兄弟节点的路径都少了1个黑，黑向上传递，如果父节点是红，父节点染黑，结束，如果父节点是黑，则父节点变为双黑节点，递归处理
        sibling->setRed();
        if(RbTreeNode::isRed(parent)){
            parent->setBlack();
            return nullptr;
        }else{
            return parent;
        }
    }
}
/*
处理双黑节点，当删除黑的叶子节点时，为了保持黑节点个数不变，假设删除的位置有一个双黑节点，2种方法将双黑变成单黑：
1、通过旋转，在双黑节点所在的路径，增加一个黑节点。
2、将兄弟节点变红，黑向上传递，如果父节点是红，父节点染黑，结束，如果父节点是黑，则父节点变为双黑节点，递归处理。
*/
void handleDoubleBlack(RbTreeNode* dBlack){
    while(dBlack!=nullptr){
        //兄弟节点一定不为空，如果为空，当前节点和兄弟节点的黑高不相等(删除前)，根据红黑树的定义，兄弟节点的黑高一定相同。
        //根据兄弟节点和侄子节点的颜色，分情况处理
        RbTreeNode* parent=dBlack->getParent();
        RbTreeNode* sibling=dBlack->getSibling();
        RbTreeNode* farNePhew=dBlack->getFarNephew();
        RbTreeNode* neiboNephew=dBlack->getNeiboNephew();
        if(RbTreeNode::isBlack(sibling)){
            dBlack=handleBlackSibling(dBlack);
        }else{
            //兄弟节点是红，先转成兄弟是黑的情况，再递归处理
            if(dBlack==parent->getLeft()){
                RRRotate(parent);
            }else{
                LLRotate(parent);
            }
            //交换父节点和兄弟的颜色
            parent->setRed();
            sibling->setBlack();
            dBlack=handleBlackSibling(dBlack);
        }
    }
}
void remove(int v){
    RbTreeNode* res=search(v,nullptr);
    //节点不存在，删除失败
    if(res==nullptr) return;
    /*
    待删除的节点，如果有2个孩子，用它的前驱或后继节点，替换它，转而删除前驱或后继节点，而这个前驱或后继节点，一定有0或1个孩子，如果有1个孩子，用这个孩子替换它，如果有0个孩子，直接删除。
    如果前驱或后继节点是黑色节点，删除它之前，将它当成双黑节点进行处理。
    */
    RbTreeNode* curr=res;
    if(curr->getLeft()!=nullptr&&curr->getRight()!=nullptr){
        //把curr转成0或1个孩子的情况
        RbTreeNode* replace=curr->getLeft();
        while(replace->getRight()!=nullptr)
            replace=replace->getRight();
        curr->setData(replace->getData());
        curr=replace;
    }
    //此时的curr，一定是0或1个孩子
    if(RbTreeNode::isBlack(curr)){
        handleDoubleBlack(curr);
    }//curr如果是红色节点，直接删除，不影响它所在路径的黑色节点数
    RbTreeNode* parent=curr->getParent();
    //包含了0个或1个孩子的情况
    RbTreeNode* child=nullptr;
    if(curr->getLeft()!=nullptr) child=curr->getLeft();
    else child=curr->getRight();
    if(curr==parent->getLeft()){
        parent->setLeft(child);
    }else{ //curr==parent->getRight()
        parent->setRight(child);
    }
    if(child!=nullptr) child->setParent(parent);
    delete curr;
}
```

### 最大堆，最小堆
定义：

+ 是一棵完全二叉树
+ 根节点的值大于左右孩子，同时左右子树也是最大堆

为什么一定要是完全二叉树：可以用数组存储，程序局部性更好，高度为h的完全二叉树的节点数n=2^(h-1) - 1 ~ 2^h - 1，高度h=O(log2n)，从而使得最大堆的查找和删除效率是O(log2n)。

**<font style="color:#DF2A3F;">数组调整成堆</font>**：已有一个数组，将它调整为最大堆，从底部的第一个非叶子节点开始，到根节点，每个节点执行下滤，因为是从后往前调整，所以执行到第i个节点时，它的左右子树已经是最大堆，当根节点下滤后，整个二叉树是最大堆。时间复杂度是O(N)，因为满二叉树的叶子节点有2^(h-1)，是接近一半的节点，而完全二叉树大部分的节点都在底层，只要做常数级的比较交换，少数需要O(logN)，总体仍是O(N)。

```cpp
/*
    向最大堆插入一个节点，常规逻辑：将节点插入数组尾部，比较插入节点和父节点的大小，如果大于父节点，交换它们的值，节点上移，继续比较节点和父节点的大小，直到小于父节点，插入结束。
    优化逻辑：交换值的时候，因为每次交换的值中，都有一个是插入节点的值，节点的值已经在临时变量中了，所以可以不把父节点的值赋值为节点的值，只需要将节点的值赋值为父节点的值，比较的时候，直接比较插入的值和父节点的值，只有当父节点的值大于插入的值的时候，才将插入的值写入父节点。
    */
    void insert(int v){
        if(sentinel) insertWithSentinel(v);
        else insertNoSentinel(v);
    }
    void insertWithSentinel(int v){
        if(isFull()) return;
        int i=size;
        while(v>this->data[i/2]){
            this->data[i]=this->data[i/2];
            i/=2;
        }
        //v<=this->data[i/2],并且i的值已经复制给了它的子节点
        this->data[i]=v;
        size++;
    }
    void insertNoSentinel(int v){
        if(isFull()) return;
        int i=this->size;
        this->size=i+1;
        while(i>0){
            int parentIdx=(i-1)>>1;
            MaxHeapNode& parent=this->data[parentIdx];
            if(v<parent) break;
            this->data[i]=parent;
            i=parentIdx;
        }
        this->data[i]=v;
    }
    /*
    删除最大堆中最大的元素，并返回它。
    常规逻辑：用数组最后一个节点替换堆顶节点，然后从堆顶开始，判断当前节点和最大子节点的大小，如果小于，交换，节点下移，循环判断，如果大于，或者已经下移到了叶子节点，删除结束。
    优化逻辑：因为每次交换的值中，都有一个是数组最后一个节点的值，可以先把它保存到临时变量，交换的时候不把最大子节点的值赋值为节点的值，只需要将节点的值赋值为子节点的值，比较的时候，直接比较临时变量的值和最大子节点的值，只有当最大子节点的值小于临时变量，或者到达叶子节点的时候，才将临时变量写入它的父节点，或者叶子节点。
    */
    int remove(){
        if(sentinel) return removeWithSentinel();
        else return removeNoSentinel();
    }
    int removeWithSentinel(){
        if(isEmpty()) return -1;
        int res=this->data[1].data;
        size--;
        int temp=this->data[size].data;
        int parent=1;
        while(true){
            //childIdx是最大子节点的索引
            int childIdx=parent*2;
            //parent是否是叶子节点
            if(childIdx>=size) break;
            if(this->data[childIdx]<this->data[childIdx+1]) 
                childIdx++;
            if(temp>this->data[childIdx]) break;
            this->data[parent]=this->data[childIdx];
            parent=childIdx;
        }
        //parent是叶子节点，或者它的最大子节点<temp
        this->data[parent]=temp;
        return res;
    }
    int removeNoSentinel(){
        if(isEmpty()) return -1;
        int res=this->data[0].data;
        size--;
        int temp=this->data[size].data;
        int parent=0;
        while(true){
            //childIdx是最大子节点的索引
            int childIdx=(parent<<1)+1;
            if(childIdx>=size) break;
            if(this->data[childIdx]<this->data[childIdx+1]) 
                childIdx++;
            if(temp>this->data[childIdx]) break;
            this->data[parent]=this->data[childIdx];
            parent=childIdx;
        }
        this->data[parent]=temp;
        return res;
    }
```

### 字典树（前缀树）
定义：字典树是一棵由节点和边组成的多叉树，每条边代表一个字符，对于字符串集合S：

+ 如果S是空集，字典树只有1个根节点。
+ 如果S不是空集，S中的每个字符串，等于一个节点的路径上，所有边的字符组成的字符串，这样的节点称为终止节点，如果节点的路径上，所有边的字符组成的字符串，不在集合S中，这样的节点称为非终止节点。

性质：如果节点的边是有序的，树的DFS遍历得到的字符串是有序的（**字典**序）。

数据结构表示：用邻接矩阵，横坐标代表节点编号，纵坐标代表边的字符，值代表节点通过边连接的另一个节点，另外需要一个数组来存储节点是否终止节点。

插入：遍历插入的字符串s中的字符，从根节点开始，根据当前字符选择对应的边，直到没有对应的边，新建x个节点和边，保存s中剩下的字符。插入的时间复杂度是O(n)，n是插入字符串的字符数。

查找：遍历查找的字符串s中的字符，从根节点开始，根据当前字符选择对应的边，直到没有对应的边，或者s中的字符遍历结束后，遇到了非终止节点，字符串s不在集合中，如果s中的字符遍历结束后，遇到了终止节点，字符串s在集合中。查找的时间复杂度是O(n)，n是查找字符串的字符数。

### 基数树
字典树存在节点数过多的问题，基数树把只有1个节点的非终止节点，合并为1个节点，边的字符合并成字符串，减少了节点个数。

定义：是一棵由节点和边组成的多叉树，每条边代表一个字符串，对于字符串集合S：

+ 如果S是空集，树只有1个根节点。
+ 如果S不是空集，S中的每个字符串，等于一个节点的路径上，每条边组成的字符串，这样的节点称为终止节点，如果节点的路径上，每条边组成的字符串，不在集合S中，这样的节点称为非终止节点。
+ 除了根节点，所有非终止节点，至少有2个子节点。

性质1：只有1个子节点的非终止节点可以合并，因为合并之后的边，仍然可以表示集合S中的字符串的前缀，有2个子节点的非终止节点不可以合并，因为合并之后的边，只能表示一个前缀串。

性质2：节点个数<=2n-1，n是集合S的字符串个数，证明如下，假设字符串个数是n，非终止节点数是x，可以得到终止节点个数是n，边的个数是节点总数-1，因为非终止节点至少对应2条边，所以边数的下界是2x。

基数树比字典树，减少了节点个数，更好的程序局部性，但空间复杂度一样，最坏情况都是所有字符串长度之和。

插入：先查找s是否已经在树中，

1. 如果在:  判断当前节点是否是压缩节点，如果是，将它分成2个节点，它们的isTerminate=true。如果不是，直接将对应字符的isTerminate=true。
2. 如果不在：

      如果当前节点是压缩节点，创建一个2个孩子的非压缩节点a，保存第一个不匹配的字符，和s的剩余第1个字符，创建一个节点b，保存s的剩余x-1个字符，孩子1指向原来的子节点，孩子2指向b。

     如果当前节点不是压缩节点，直接将比较之后的剩余字符加到它的子节点：判断s去掉已匹配的前缀之后的字符个数，如果是1，直接添加到当前节点，如果大于1，新建一个节点，将第一个字符保存到当前节点，剩下的字符保存到新建的节点，当前节点新增一个孩子，指向新建节点。

```cpp
void insert(const char str[],int len){
    char s[len];
    memcpy(s,str,len);
    RadixTreeNode *last=nullptr;
    RadixTreeNode** parent=nullptr;
    int splitIdx=0, sIndex=0;
    find(s, len,&parent, &last, &splitIdx, &sIndex);
    if(sIndex==len){
        //s已经在树中,last指向路径是s的节点
        //TODO 还需要判断当前节点是否是压缩节点
        last->setTerminate(splitIdx, true);
    }else if(last==nullptr){
        //s还有后缀没有匹配，树已经遍历到了叶子节点
        RadixTreeNode* com=new RadixTreeNode(&s[sIndex],len-sIndex,nullptr,true);
        *parent=com;
    }
    else if(!last->isCompress()){
        //如果当前节点不是压缩节点，直接将比较之后的剩余字符加到它的子节点：s去掉已匹配的前缀之后的字符，其中第一个字符，直接添加到当前节点，剩下的字符保存到新建的节点，当前节点新增一个孩子，指向新建节点。
        RadixTreeNode* child=nullptr;
        if(sIndex+1<len){//sIndex=7 len=10
            child=new RadixTreeNode(&s[sIndex+1],len-sIndex,nullptr,true);
        }
        last=RadixTreeNode::appendChild(last, child, s[sIndex]);
        *parent=last;
    }else{
        //当前节点是压缩节点，创建一个2个孩子的非压缩节点nonComp，保存第一个不匹配的字符，和s的剩余第1个字符，创建一个压缩节点com，保存s的剩余x-1个字符，nonComp的孩子1指向原来的子节点，孩子2指向b。

        //2个孩子的非压缩节点
        RadixTreeNode* twoChildren=new RadixTreeNode(2);
        twoChildren->setData(0,last->getData(splitIdx));
        twoChildren->setData(1,s[sIndex]);
        //0~splitIdx
        RadixTreeNode* newChildComp=twoChildren;
        if(splitIdx>0)
            newChildComp=new RadixTreeNode(last->getData(),splitIdx,twoChildren,false);
        *parent=newChildComp;
        splitIdx++;
        sIndex++;
        int origiCompSize=last->getSize();
        if(splitIdx<origiCompSize){
            //压缩节点剩余的字符
            RadixTreeNode* com=new RadixTreeNode(last->getData()+splitIdx,origiCompSize-splitIdx,last->getChildren(0),last->isTerminate(0));
            twoChildren->setChildren(0, com);
        }else{
            twoChildren->setTerminate(0, true);
        }
        if(sIndex<len){
            //s剩余的字符
            RadixTreeNode* comp=new RadixTreeNode(&s[sIndex],len-sIndex,nullptr,true);
            twoChildren->setChildren(1, comp);
        }else{
            twoChildren->setTerminate(1, true);
        }
        delete last;
    }
}
```

### 并查集
定义：是一种用于快速判断2个元素是否处于同一集合，以及集合并运算的数据结构。不关心集合元素间的具体关系，只关心元素是否在集合中。

具体实现：用树来存储集合元素，树使用双亲表示法，节点保存父节点指针，不保存孩子节点指针，通过根节点是否相同来判断是否处于同一集合。使用数组实现双亲表示法，数组下标是节点的编号，数组元素是父节点在数组中的编号，根节点下标为负数，绝对值是集合元素个数，2个集合并运算，将1个集合的根节点的父节点指针指向另一个集合，从而树的高度最多加1，用元素个数少的集合并元素个数多的集合，防止新增1个元素时，树退化成链表。如下，新增元素的顺序是1,2,3，3和1-2合并时，如果是大集合并到小集合，会导致如下情况。

```cpp
3
 \
  1
   \
    2
```

并查集代码实现如下，

```cpp
//查询数据所在的并查集的根节点的编号,idx是节点编号
int FindUf(int uf[],int idx){
    int pre=idx;
    while((idx=uf[pre])>-1){
        pre=idx;
    }
    return pre;
}
//将2个数据所在的并查集合并,如果它们属于同一集合，不合并，否则合并
void UnionUf(int uf[],int idx1,int idx2){
    int root1=FindUf(uf,idx1);
    int root2=FindUf(uf,idx2);
    if(root1==root2) return;
    //获取元素个数更多的和更少的集合的根节点索引，根节点是负数，更小的绝对值更大
    int more=root1,less=root2;
    if(uf[root1]>uf[root2]) {
        more=root2;
        less=root1;
    }
    uf[more]+=uf[less];//负数相加，等于绝对值相加的相反数
    uf[less]=more;
}
//判断2个节点是否属于同一个并查集，如果是，返回1，如果不是，返回0
int IsSameUf(int uf[],int idx1,int idx2){
    //根节点的索引相同
    return FindUf(uf,idx1)==FindUf(uf,idx2);
}
```

# 图
## 定义
定义：n个元素的集合，元素之间是多对多的关系。

由顶点和边组成，顶点代表集合中的元素，边代表元素之间的关系，也就是顶点对，比如顶点{A,B,C,D}，边(A,B),(A,C)...，有向边：顶点对表示单向的关系，无向边：顶点对表示双向的关系。

图由G(V,E)表示，V表示顶点的集合，E表示边的集合。

无向图：由顶点和无向边组成的图。

有向图：由顶点和有向边组成的图。

完全图：任意2个顶点有1条边的图，边的个数达到最大值。

带权图：每条边对应一个权重。

无权图：边没有权重。

稠密图：边的个数较多，和顶点个数的平方一个数量级，边最大等于|V|*(|V|-1)。

稀疏图：边的个数较少，和顶点个数一个数量级。

## 图的遍历
深度优先遍历（DFS，Depth）：从任意一个顶点v开始，访问v，然后取v的一个未访问过的邻接点，对它进行深度优先遍历。类似于树的先序、中序、后序遍历，取决于访问v的时机，时间复杂度：邻接表，每个顶点恰好访问一次，判断每个顶点的邻接点时，判断次数等于边的个数，所以是O(|V|+|E|)，邻接矩阵，每个顶点恰好访问一次，判断每个顶点的邻接点时，需要判断所有其他顶点是否是邻接点，所以是O(|V|^2)。

![](/assets/images/1774949654405-23fcd5ad-7edf-4ad3-a23d-0f46892dc2eb.webp)

广度优先遍历（BFS，Breath First Search）：从任意顶点v开始，先访问v，然后将v的<font style="color:#DF2A3F;">未入过队</font>邻接点加到队列，对队列中的每个元素进行广度优先搜索。类似于树的层序遍历，时间复杂度：邻接表，每个顶点恰好访问一次，判断每个顶点的邻接点时，判断次数等于边的个数，所以是O(|V|+|E|)，邻接矩阵，每个顶点恰好访问一次，判断每个顶点的邻接点时，需要判断所有其他顶点是否是邻接点，所以是O(|V|^2)。

区别：

1. DFS是优先往深处遍历，直到没有未访问的邻接点时，再回溯，BFS是一层一层向外扩展。BFS遍历到顶点时，是v到它的最短路径，适合查找最短路径。
2. DFS需要visited数组来记录每个顶点是否已经访问，因为进栈的同时访问顶点，回溯的时候需要获取顶点未访问过的邻接点，BFS需要enQueued数组来记录每个顶点是否已经入队，因为BFS入队的时候，不会马上访问。

## 图的连通性
路径：如果存在集合{V，v1,v2...vn，W}，其中V,Wvi是图中的顶点，集合中任意2个相邻顶点存在1条边，则该集合是V和W的路径。

路径的长度：对于无权图，是路径的边的个数，对于带权图，是路径的边的权重和。

简单路径：vi互不相同。

回路：起点等于终点的路径。

连通：如果顶点v和w存在1条路径，则v和w是连通的。

连通图：无向图的任意2个顶点连通。

连通分量：极大连通子图。达到极大顶点数，再加一个就不连通了，极大边数：子图所有顶点相连的边。

## 最短路径问题
最短路径：在图中，V和W之间所有的路径，边的权值最小的路径，是V和W的最短路径，第一个点是源点，最后一个点是终点。

单源最短路径：源点是固定的一个点，求它到其他所有顶点的最短路径。

多源最短路径：源点是图的每个顶点，求它们到其他所有顶点的最短路径。

无权图的单源最短路径：从源点v开始，BFS遍历，每经过一层，路径长度+1。因为无权图中，v到顶点的最短路径长度，等于v遍历到顶点的层数。

有权图的单源最短路径：Dijkstra算法，假设图的顶点集合是V，源点v，集合S是确定了v的最短路径的顶点的集合，对于顶点u∈V-S，dist[u]是v经过S中的顶点到u的最短路径长度，则u∈V-S，dist[u]的最小值对应的u，就是v到u的最短路径，所以可以将u加到S，然后更新V-S集合中顶点的dist值。

算法直觉1：假设u∈V-S，dist[u]的最小值对应的u，不是v到u的最短路径，那么存在一条更短的路径{v,...x,y...u}，其中y是V-S中的顶点，得到dist[y]<dist[u]，与dist[u]是最小值矛盾，所以假设不成立。

算法直觉2：将u加到S后，需要更新u在V-S的邻接点的dist值，对于邻接点w，dist[w]=min{dist[w], dist[u]+weight[u,w]}，取u∈V-S，dist[u]的最小值对应的u时，u一定是S中的顶点的邻接点。

边的权值必须是非负数，如果有负数权值，直觉1不成立，存在更短的路径{v,...x,y...u}，其中y是V-S中的顶点，y到u的路径权值是负数，使得dist[y]>dist[u]。

时间复杂度：如下，如果直接遍历每个顶点，计算dist最小值，需要|V|^2，计算dist的值，需要|E|，总共O(|V|^2+|E|)，如果使用最小堆计算dist最小值，需要|V|*log|V|，计算dist值之后，需要调整最小堆，需要|E|*log|V|，总共O(|V|*log|V|+|E|*log|V|)

多源最短路径：Floyd算法，定义Dk[i][j]是i经过v0~vk到j的最短路径，D0[i][j]表示i经过v0到j的最短路径，初始值D-1[i][j]是图的邻接矩阵，由Dk-1[i][j]推出Dk[i][j]，分情况讨论：

1. 当i经过v0~vk到j的最短路径，不经过k时，也就是经过v0~vk-1，Dk[i][j]=Dk-1[i][j]。
2. 当i经过v0~vk到j的最短路径，经过k时，路径是{i,...k...j}，Dk[i][j]=Dk-1[i][k]+Dk-1[k][j]，i到k的路径，一定不包括k，所以是Dk-1[i][k]。

综上，Dk[i][j]=min{Dk-1[i][j], Dk-1[i][k]+Dk-1[k][j]}。

Floyd算法能够处理负数权重，但不能处理负权的环，有负权环时，经过它的最短路径是负无穷，而Floyd算法得到的是一个有限的负数，无向图的边是双向的，只要有负数权重的边，就是负权环。

Dijkstra算法，最小堆+邻接表实现，代码如下

```cpp
/*
带权图单源最短路径，Dijkstra算法，最小堆+邻接表实现。
时间复杂度O(VlogV+ElogV)=O((V+E)logV)，V是需要遍历每个顶点，E是获取邻接点，logV是插入、删除最小堆，适用于稀疏图。
paths:返回值，每个顶点最短路径的前一个顶点。
dists:返回值，每个顶点到源点的最短路径长度。
*/
void Dijkstra(WeightGraphAdjList g,int source,int paths[],int dists[]){
    int vertexN=g->vertexN;
    WeightGraphAdjListNode* vertexes=g->vertexes;
    //初始化最小堆，用来存储V-S集合中的路径，并且获取V-S中的路径的最小值
    //最小堆中有可能存储了重复顶点的不同路径
    PdMh mh=CreatePdMh(vertexN*vertexN,ComparePathData);
    //初始化V-S集合：将源点加入最小堆
    InsertPdMh(mh,CreatePathData(source,0));
    //VS:0表示不在V-S集合，1表示在V-S集合
    char VS[vertexN];
    memset(paths,-1,sizeof(int)*vertexN);
    memset(VS,1,vertexN);
    //将dists初始化为一个很大的正数，避免将int的符号位置1，导致变成负数
    memset(dists,0x3f,sizeof(int)*vertexN);
    dists[source]=0;
    while(!IsEmptyPdMh(mh)){
        //从V-S中取一路径最小值的顶点v，它就是最短路径，然后遍历v的邻接点，比较路径大小，更新它们经过S集合到源点的路径
        //使用最小堆实现V-S时，更新路径比较麻烦，不直接更新路径，而是直接将新的路径加入到最小堆，取最短路径时，需要过滤掉已经加入S集合的顶点(S集合中，不为-1的)
        PathData shortest=RemovePdMh(mh);
        if(VS[shortest->idx]){
            VS[shortest->idx]=0;
            WeightGraphAdjListNode adj=vertexes[shortest->idx];
            while(adj){
                int currDist=shortest->dist+adj->weight;
                InsertPdMh(mh,CreatePathData(adj->vertex, currDist));
                if(currDist<dists[adj->vertex]){
                    dists[adj->vertex]=currDist;
                    paths[adj->vertex]=shortest->idx;
                }
                adj=adj->next;
            }
        }
    } 
}
```

Dijkstra算法，遍历+邻接矩阵实现，代码如下

```cpp
/*
Dijkstra算法，遍历+邻接矩阵实现。
时间复杂度：O(V^2+V^2)=O(V^2)，遍历每个顶点是V，每个顶点遍历V-S集合获取路径最小值是V，每个顶点更新邻接点路径是V,适合稠密图
*/
void DijkstraPlain(GraphAdjMat g,int source,int paths[],int dists[]){
    int vertexN=g->vertexNum;
    int* weights=g->datas;
    //path表示最短路径的前一个顶点,dists是最终最短路径长度
    char VS[vertexN];
    memset(paths,-1,sizeof(int)*vertexN);
    memset(VS,1,vertexN);
    memset(dists,0x3f,sizeof(int)*vertexN);
    dists[source]=0;
    for(int i=0;i<vertexN;i++){
        //取V-S中的路径最小值的顶点v,放到S集合，同时更新v的邻接点距离源点的路径
        int shortest=0;
        for(shortest=0;shortest<vertexN;shortest++){
            if(VS[shortest])break;
        }
        for(int j=shortest+1;j<vertexN;j++){
            if(VS[j]&&dists[j]<dists[shortest]){
                shortest=j;
            }
        }
        VS[shortest]=0;
        for(int adj=0;adj<vertexN;adj++){
            if(VS[adj]){
                int w=MatrixGet(weights,shortest,adj);
                //权值大于0，是邻接点
                int d; 
                if(w>0&&(d=w+dists[shortest])<dists[adj]){
                    dists[adj]=d;
                    paths[adj]=shortest;
                }
            }
        }
    }
}
```

Floyd算法，邻接矩阵实现，代码如下

```cpp
/*
带权图多源最短路径，Floyd算法，邻接矩阵实现。
paths: 二维数组，每行顶点作为源点，到其他顶点最短路径的上一个顶点。
dists: 矩阵下三角，每个顶点作为源点，到其他顶点最短路径的长度。
6 7
0 1 10
0 2 2
0 3 8
1 5 2
2 3 3
3 5 1
3 4 5
*/
void Floyd(GraphAdjMat g,int paths[],int dists[]){
    int *vertexes=g->datas;
    int vertexN=g->vertexNum;
    memset(paths,-1,sizeof(int)*vertexN*vertexN);
    //用邻接矩阵初始化dists
    for(int i=0;i<vertexN;i++){
        for(int j=0;j<vertexN;j++){
            int w;
            if(i!=j){
                if((w=MatrixGet(vertexes,i,j))){
                    //邻接点，dists初始化为权值,paths初始化为邻接点
                    *MatrixGetAddr(dists, i, j)=w;
                    paths[i*vertexN+j]=i;
                }
                else
                    //非邻接点，初始化为一个很大的正数，但不能初始化为INT_MAX，避免溢出
                    *MatrixGetAddr(dists, i, j)=0xffff;
            }
        }
    }
    for(int k=0;k<vertexN;k++){
        for(int i=0;i<vertexN;i++){
            for(int j=0;j<vertexN;j++){
                if(i!=j){
                    int passK=MatrixGet(dists, i, k)+MatrixGet(dists, k, j);
                    int notPassK=MatrixGet(dists, i, j);
                    *MatrixGetAddr(dists, i, j)=passK<notPassK?passK:notPassK;
                    if(passK<=notPassK){
                        //最短路径经过k，i~k的最短路径已经在第i行了，只需要将第k行的k~j的路径复制到第i行
                        // printf("修改前:\n");
                        // PrintSquareMat(paths, vertexN);
                        int curr=j,pre=j;
                        while((pre=paths[k*vertexN+curr])!=-1){
                            paths[i*vertexN+curr]=pre;
                            curr=pre;
                        }
                        // printf("修改后:\n");
                        // PrintSquareMat(paths, vertexN);
                    }
                }
            }
        }
    }
}
```

## 最小生成树
定义：对于图G(V,E)的最小生成树，，是一棵有|V|个顶点、|V|-1条边、顶点和边都在图中、没有回路、边的权重和最小的树。

性质1：最小生成树中加入任意一条边，将形成回路。

性质2：如果最小生成树存在，则图一定是连通图。

得到最小生成树的算法：Prim算法和Kruskal算法。

Prim算法：是一种贪心算法，每次取未加入集合S的、距离集合S的顶点的权重最小者，类似于Dijkstra算法，dist[u]的计算不同，改成了距离集合S的顶点的权重最小值，parent数组用来保存树的子节点到父节点的映射，如下

<!-- 这是一张图片，ocr 内容为： -->
![](/assets/images/1774961945303-1d03c5e3-c700-4fe9-b485-df0fdf37742c.webp)

Prim算法的证明：除了第一个顶点，每次取一个顶点和一条边加入生成树，所以最终一定V-1条边，是生成树。假设当前集合S和V-S，S到V-S最小权重的边是e1，最终的最小生成树是T，如果e1∈T，证明结束，如果e1不属于T，将e1加入T一定会成环，环路中，有e2和e1将S和V-S连通，而e2权值大于e1，T不是最小生成树，反证e1∈T。

Prim算法适合稠密图，时间复杂度和顶点个数有关，是O(V^2)，外层的V表示每个顶点取一次，内层的V表示更新邻接点、取最小权重。

Kruskal算法如下，每次取权重最小的边，然后判断是否构成回路。适合稀疏图，时间复杂度和边的个数有关，是O(E+ElogE)=O(ElogE)，E+ElogE的E是构建最小堆，E是取的边数，logE是从最小堆取一条边。

Prim算法代码如下

```cpp
/*
最小生成树Prim算法，邻接矩阵实现。
假设集合S是最小生成树，初始为任意某个顶点，每次从V-S中，取距离S的顶点权重最小的顶点，加到S，更新V-S顶点的权重。
MST:返回值，最小生成树，矩阵下三角。
6 7
0 1 10
0 2 2
0 3 8
1 5 2
2 3 3
3 5 1
3 4 5
*/
void Prim(GraphAdjMat g,int parent[]){
    int *vertexes=g->datas,V=g->vertexNum;
    //初始化集合V-S
    char VS[V];
    int shortest=0,dists[V],mstLen=1;
    memset(VS, 1, V);
    memset(dists,0x3f,sizeof(int)*V);
    memset(parent, -1, sizeof(int)*V);
    while(1){
        VS[shortest]=0;
        //遍历shortest在V-S的邻接点，更新S到V-S的路径权重
        int adj=0,noAdd=1;
        for(;adj<shortest;adj++){
            int w=MatrixGetRaw(vertexes,shortest,adj);
            if(VS[adj]&&w!=0){//在V-S集合中的邻接点
                if(w<dists[adj]) {
                    dists[adj]=w;
                    parent[adj]=shortest;
                    mstLen++;
                    noAdd=0;
                }
            }
        }
        adj++;//跳过shortest
        for(;adj<V;adj++){
            int w=MatrixGetRaw(vertexes,adj,shortest);
            if(VS[adj]&&w!=0){//在V-S集合中的邻接点
                if(w<dists[adj]) {
                    dists[adj]=w;
                    parent[adj]=shortest;
                    mstLen++;
                    noAdd=0;
                }
            }
        }
        //包括了V-S变为空集、S和V-S不连通，这2种情况
        if(noAdd) break;
        //取dists中在V-S中最小的
        shortest=0;
        for(;shortest<V;shortest++){
            if(VS[shortest]) break;
        }
        for(int j=shortest+1;j<V;j++){
            if(VS[j]&&dists[shortest]>dists[j]) shortest=j;
        }
    }
    if(mstLen<V){
        printf("不存在最小生成树\n");
    }
}
```

KrusKal算法代码如下

```cpp
/*
最小生成树Kruskal算法。
假设边的集合是E，每次从E取权重最小的边，加到生成树，判断是否有环路，如果有，将边从E删除，继续从E取下一个权重最小的边。
MST：是返回值，最小生成树的邻接矩阵表示。
*/
void Kruskal(GraphAdjMat g,int MST[]){
    int *vertexes=g->datas,V=g->vertexNum;
    EdgeMh mh=CreateEdgeMh(16, CompareEdgeFun);
    //遍历图的每条边，初始化最小堆
    for(int col=0;col<V;col++){
        for(int row=col+1;row<V;row++){
            int w=vertexes[row*(row-1)/2+col];
            if(w!=0)
                InsertEdgeMh(mh,CreateEdge(w, row, col));
        }
    }
    Edge smallest;
    //并查集，用来判断是否有环路，生成树中的顶点属于同一集合，判断环路，就是判断新加入的边的2点，是否已经属于同一生成树
    int uf[V],v1,v2,edgeNum=0;
    memset(uf,-1,sizeof(int)*V);
    memset(MST,0,V*(V-1)/2*sizeof(int));
    while(!IsEmptyEdgeMh(mh)&&edgeNum<V-1){
        smallest=RemoveEdgeMh(mh);
        v1=smallest->v1,v2=smallest->v2;
        if(!IsSameUf(uf,v1,v2)){
            //新加入的边没有形成环路，将2个顶点所在的并查集合并，表示形成一个更大的生成树
            UnionUf(uf, v1, v2);
            //加入到最小生成树
            MST[v1*(v1-1)/2+v2]=smallest->w;
            // PrintMST(MST,V);
            edgeNum++;
        }
    }
    if(edgeNum<V-1)
        printf("最小生成树不存在\n");
}
```

## 拓扑排序
拓扑序：有向无环图（DAG）的顶点序列(v1,v2...vi...vj...vn)，其中任意2点vi和vj，如果vi到vj存在一条有向路径，则vi一定排在vj前面，这样的序列是有向无环图的一个拓扑序。拓扑序不只一个。

拓扑排序：获取拓扑序的过程，如下，初始时，记录每个顶点的入度，每次取入度为0的顶点，输出到序列，并且将顶点的邻接点的入度减1，循环取下一个入度为0的顶点。

![](/assets/images/1775735183096-dd930650-a7d4-490e-9515-4610c1255eb5.webp)

拓扑排序代码如下，邻接矩阵实现，时间复杂度O(V^2)，因为需要获取邻接点，如果使用邻接表，时间复杂度是O(E)，拓扑排序可以用来判断有向图是否有环路，但不能得到环路的路径，并查集用于判断无向图是否有环路，但不能得到环路的路径。

```cpp
/*
拓扑排序，邻接矩阵实现，时间复杂度O(V^2)，因为需要获取邻接点，如果使用邻接表，时间复杂度是O(E)。
*/
void TopSort(DGAdjMat g,int topSeq[]){
    int *vertexes=g->vertexes,V=g->V;
    //inDegree是每个顶点的入度，每一列的1的个数，是顶点的入度，writeIdx是topSeq要写入的位置索引,readIdx是topSeq要读取的位置的索引
    int inDegree[V],writeIdx=0,readIdx=0;
    memset(inDegree, 0, sizeof(int)*V);
    memset(topSeq,-1,sizeof(int)*V);
    for(int col=0;col<V;col++){
        for(int row=0;row<V;row++){
            inDegree[col]+=vertexes[row*V+col];
        }
        if(inDegree[col]==0){
            //初始化topSeq
            topSeq[writeIdx++]=col;
        }
    }
    int v;
    while(writeIdx<V&&readIdx<writeIdx){
        //从readIdx中读取一个顶点v，将readIdx指向下一个，将v的邻接点w的入度减1，如果减为0，将w写入topSeq
        v=topSeq[readIdx++];
        for(int w=0;w<V;w++){
            if(vertexes[v*V+w]!=0&&--inDegree[w]==0){
                topSeq[writeIdx++]=w;
            }
        }
    }
    if(writeIdx<V&&readIdx==writeIdx){
        printf("图中有环路,拓扑序不存在\n");
    }
}
```

## 关键路径
是拓扑排序的应用，用DAG来表示项目的每个活动完成的先后顺序、开始时间，边表示一个活动，边的权重是活动的持续时间，顶点表示入边对应的活动的结束，和出边对应的活动的开始，顶点的属性包括：编号、最早开始时间、最晚开始时间，开始顶点的最早开始时间=最晚开始时间=0，如下图，Earliest[i]是顶点i的最早开始时间，C是活动持续时间，等于入边的Earliest+C的最大值。结束顶点的最早开始时间=最晚开始时间，Latest[i]是顶点i的最晚开始时间，等于出边的Latest-C。活动的机动时间表示执行活动时，有空闲的时间，D是机动时间，等于后置活动的最晚开始时间-当前活动的最早开始时间-C。关键路径是DAG中，机动时间等于0的活动组成的路径，这条路径的活动，如果延误，一定会导致整个项目延期。

# 排序算法
## 冒泡排序
如果要得到从小到大的序列，依次比较两两相邻的2个元素，如果前面的大于后面的，交换2个元素的值，第1轮将前n个元素的最大值移动到第n个位置，第2轮将前n-1个元素的最大值，也就是n个元素的次最大值，移动到第n-1个位置，直到第n-1轮，将前2个元素的最大值移动到第2个位置。

是稳定的排序，因为比较2个相邻的元素，如果相等，不交换值，保持原来的顺序。

优化：当某一轮比较两两相邻元素时，没有发生交换，前i个元素的子序列已经有序，可以提前结束循环。

时间复杂度：最好情况下，已经是有序序列，只需要比较第一轮，是O(N)。最坏情况下，正好逆序，O(N^2)，平均O(N^2)。

代码如下

```cpp
//冒泡排序
void Bubble_sort(int arr[],int N){
    //比较两两相邻的2个元素，将较大者往后移动，i从N-1~1，每一轮将前i个元素序列的最大值移动到前缀序列的末尾
    int noSwap;
    for(int i=N-1;i>=1;i--){
        noSwap=1;
        for(int j=0;j<i;j++){
            if(arr[j]>arr[j+1]){
                Swap(&arr[j],&arr[j+1]);
                noSwap=0;
            }
        }
        //当某一个前缀子序列已经有序时，整个序列已经有序
        if(noSwap) break;
    }
}
```

## 插入排序
将第i个元素插入到前i-1个元素的有序序列中，先暂存第i个元素，然后将第i个元素和它前面的有序序列的元素j，从后往前依次比较，当j的值大于i的值时，将j元素往后移动一位，继续比较j-1元素，直到某个元素k的值小于等于i，将i元素复制到k+1的位置。本质上也是比较和交换相邻的2个元素，但序列是有序的，只需要将i元素和前面的有序子序列交换，和冒泡排序相比，每轮只进行一次交换。

是稳定的，因为本质上也是比较和交换相邻的2个元素，相等时，不交换。

时间复杂度：最好情况下，有序序列，需要比较N-1次，O(N)，最坏情况下，逆序序列，需要比较1+2+...+N-1=O(N^2)。

```cpp
//插入排序
void Insertion_sort(int arr[],int N){
    //i从1~N-1，将第i个元素插入0~i-1这个有序序列，先暂存第i个元素，j从i-1~0，比较tmp,j，如果j>tmp，将j后移一个位置，当j<=tmp，将tmp插入到j的后面（此时j+1已经后移一位）
    //注意必须暂存i的值，因为j后移时，会覆盖i，判断i,j大小也必须使用tmp值
    for(int i=1;i<=N-1;i++){
        int j,tmp=arr[i];
        for(j=i-1;j>=0&&arr[j]>tmp;j--)
            arr[j+1]=arr[j];
        arr[j+1]=tmp;
    }
}
```

## 希尔排序
排序本质上是消除逆序对，基于比较交换的排序，如果要减小时间复杂度，一次比较交换，需要能够消除多个逆序对，比较交换相邻的元素，恰好能消除一个逆序对，比较交换不相邻的元素，能够消除多个逆序对，证明：假设序列50..100...10...40..30，交换50和50，50前面的元素和30后面的元素，和50~30的序列中的元素，相对位置不变，不改变逆序对个数，对于100，它大于50，交换后会消除一个逆序对100,30，产生一个逆序对100,50，对于10，交换后会消除一个逆序对50,10，产生一个逆序对10,30，对于10，会消除2个逆序对，加上50,30这个逆序对，至少能消除1个逆序对。

希尔排序：使用一个增量序列Dm,Dm-1...D1=1，执行m轮插入排序，第i轮，将每间隔Di个元素的序列执行插入排序，间隔的值逐渐递减，最后一轮间隔为1，执行朴素的插入排序。

为什么Dm-1排序后，Dm的序列仍然是有序的：假设任意元素xi，序列已经gap=5有序，xi<xi+5，执行gap=3排序，如果xi后面的元素xj交换到xi的位置，那么一定xj<xi<xi+5，仍然gap=5有序，如果xi前面的元素xi-3交换到xi，，xi-3>xi，如果xi-3>xi+5，那么xi+2>xi-3>xi+5，xi+2会交换到xi+5，仍然满足gap=5有序。

间隔序列如果不互质，为什么会导致排序性能下降：比如间隔9,6，任意元素xi理论上能移动的距离是9k+6m=3*(3k+2m)，只能移动3的整数倍的距离，最多只能确保gap=3有序，如果x,y互质，xk+ym能得到任意整数，因为存在xk'+ym'=1。

是不稳定的，交换不相邻的元素x,y时，x,y之间的元素如果和x,y任意一个相等，会导致相等元素的原始顺序发生变化。

```cpp
//希尔排序
void Shell_sort(int arr[],int N){
    //是插入排序的优化，使用增量序列，每次对增量序列中，间隔的元素执行插入排序
    int temp,j;
    for(int gap=N/2;gap>=1;gap/=2){
        //i从gap开始，将i插入[0,i-gap]范围内的前缀间隔序列
        for(int i=gap;i<N;i++){
            temp=arr[i];
            for(j=i-gap;j>=0&&temp<arr[j];j-=gap)
                arr[j+gap]=arr[j];
            arr[j+gap]=temp;
        }
    }
}
```

堆排序是一种选择排序，使用堆来获取最小元，朴素的堆排序：将待排序列复制到数组中，将数组调整为最小堆，取堆顶元素写回原序列，时间复杂度是O(N+NlogN)=O(NlogN)，空间复杂度是O(N)。

优化的堆排序：如果要得到从小到大的序列，先将原数组调整成最大堆，然后执行N-1轮循环，每一轮处理前i个元素的子序列，将堆顶的子序列最大值，和子序列最后一个元素交换，将子序列的最大值放到正确的位置，堆顶元素执行下滤。时间复杂度是O(N+NlogN)=O(NlogN)，空间复杂度是O(1)。如下

是不稳定的，因为下滤的过程中，可能会交换不相邻的元素。

```cpp
void Heap_sort(int arr[],int N){
    //先构建最大堆,i是子序列最后一个元素，从N-1到0，堆顶是子序列最大值，和i交换,i换到堆顶后，执行下滤
    BuildMaxHeap(arr,N);
    for(int i=N-1;i>=0;i--){
        Swap(&arr[0],&arr[i]);
        DownMaxHeap(arr,i,0);//注意子序列的长度减1
    }
}
//将数组调整为最大堆
void BuildMaxHeap(int arr[],int N){
    //获取从后往前数，第一个非叶子节点，也就是最后一个节点的父节点,(N-2)/2，从它开始，往前遍历，到根节点，执行下滤
    for(int i=(N-2)/2;i>=0;i--){
        DownMaxHeap(arr,N,i);
    }
}
//最大堆的下滤,以top为堆顶
void DownMaxHeap(int arr[],int N,int top){
    int parent=top,child,tmp=arr[parent];
    while((child=(parent<<1)+1)){
        if(child>=N) break;
        if(child+1<N&&arr[child]<arr[child+1]) child++;
        if(tmp>=arr[child]) break;
        arr[parent]=arr[child];
        parent=child;
    }
    arr[parent]=tmp;
}
```

## 归并排序
递归实现：将序列分成2个等长的子序列，对子序列执行归并排序，合并这2个有序的子序列，递归终止条件：当子序列长度为1时，终止递归。

非递归：如下，先每2个元素合并，得到长度为2的子序列，再将长度为2的子序列合并，得到长度为2^2的子序列，直到合并成长度为N的序列。

时间复杂度：归并的轮数近似是一棵满二叉树的高度，O(logN)，每轮归并，比较和复制的次数是O(N)，所以O(NlogN)，最好情况和最坏情况相同。

空间复杂度：O(N)，因为合并2个有序子序列，需要额外空间保存合并结果。多用于外部排序，比如数据库。

是稳定的，因为合并有序子序列时，如果相等，当成左子序列的值更小来处理，从而保持原始顺序。

```cpp
void Merge_sort_rec(int arr[],int N){
    //将序列分成等长的2部分，对它们进行归并排序，得到有序的序列，将它们合并到一个临时数组，再复制回原数组
    int temp[N];
    DoMergeSortRec(arr,0,N,temp);
}
void DoMergeSortRec(int arr[],int begin,int end,int temp[]){
    if(end-begin==1) return;
    int middle=(begin+end)/2;
    DoMergeSortRec(arr, begin, middle,temp);
    DoMergeSortRec(arr, middle, end,temp);
    Merge(arr,temp,begin,middle,end);
}
//将arr的2个有序序列，[left,right)和[right,end)，合并到temp的[left,end)，然后将temp的[left,end)复制回arr的[left,end)
void Merge(int arr[],int temp[],int left,int right,int end){
    int tempI=left,i=left,j=right;
    for(;i<right&&j<end;){
        if(arr[i]<=arr[j])
            temp[tempI++]=arr[i++];
        else
            temp[tempI++]=arr[j++];
    }
    while(i<right) temp[tempI++]=arr[i++];
    while(j<end) temp[tempI++]=arr[j++];
    for(tempI=left;tempI<end;tempI++)
        arr[tempI]=temp[tempI];
}
```

```cpp
//归并排序,非递归实现
void Merge_sort(int arr[],int N){
    //子序列长度是1,2,2^2,2^3...
    int length=1,temp[N];
    while(length<N){
        //每一轮，现将arr归并到temp，再将temp归并到arr，避免合并时将temp复制回arr，如果归并次数是2的整数倍，正好最后一次归并到arr，如果归并次数不是2的整数倍，会多执行一次复制
        DoMergeSort(arr,temp,length,N);
        length*=2;//这里length如果>=N，N除以2*length的余数等于N，正好执行Do_Merge_sort的剩余部分复制操作，将temp复制回arr
        DoMergeSort(temp,arr,length,N);
        length*=2;
    }
}
//对arr中每2个子序列，合并到temp，子序列长度是length
void DoMergeSort(int arr[],int temp[],int length,int N){
    //每轮循环处理i为起始点的2个长度为length的子序列，当N能整除2*length时，当i==N时退出循环,当N不能整除2*length时，N-2*length是最后一段2*length中的某个点，当i等于N-余数时，退出循环，i停在余数的起始位置
    int i=0;
    for(;i<=N-2*length;i+=2*length){
        Merge1(arr,temp,i,i+length,i+2*length);
    }
    if(i==N) return;//N能整除2*length
    if(N>length+i)//N-i>length
        //余数大于length，需要将length和剩余部分合并
        Merge1(arr,temp,i,i+length,N);
    else{
        //余数小于length，已经是有序的，直接将[i,N)复制到temp，因为前面的归并轮次已经合并过了，所以有序
        for(;i<N;i++) temp[i]=arr[i];
    }
}
//将arr的2个有序序列，[left,right)和[right,rightEnd)，合并到temp的[left,rightEnd)
void Merge1(int arr[],int temp[],int left,int right,int rightEnd){
    int tempI=left,i=left,j=right;
    for(;i<right&&j<rightEnd;){
        if(arr[i]<=arr[j])
            temp[tempI++]=arr[i++];
        else
            temp[tempI++]=arr[j++];
    }
    while(i<right) temp[tempI++]=arr[i++];
    while(j<rightEnd) temp[tempI++]=arr[j++];
    //不需要复制回arr
}
```

## 快速排序
从待排序列中，选一个主元，主元尽量选择接近序列平均值的元素，先将主元交换到序列最右侧，然后令i指向序列最左侧元素，j指向主元左侧元素，如果i指向的元素小于主元，向右移动i，直到i指向第一个大于等于主元的元素，如果j指向的元素大于主元，向左移动j，直到j指向第一个小于等于主元的元素，交换i和j指向的元素，循环移动i和j，直到i>j，交换i和主元，递归对主元左右2侧的序列做快速排序，当序列长度变为1时，递归结束。

主元选择为什么要接近均值：能够得到2个大小接近的子序列，子序列长度更快减小为1，从而减小递归层数。

Median3算法：选择序列左、中、右的元素中位数，具体做法是，对左、中、右3个元素做插入排序，使左<=中<=右，再将中交换到右，返回中。

如果i或j等于主元，为什么要停下来交换，而不是继续移动：当序列中的元素全都和主元相等时，如果继续移动，会导致i移动到最右侧，主元最终在序列的最右侧，得到的2个子序列大小严重不平衡，子序列需要O(N)时间减为1，整体时间复杂度退化为O(N^2)。而如果停下来交换，主元能够在序列靠近中心的位置，得到的2个较为平衡的子序列。

时间复杂度：取决于主元的选择，最好情况下，每次主元都能选到接近中心的位置，O(NlogN)，最坏情况下，每次主元选到序列的最值，O(N^2)，平均O(NlogN)。

快速排序的优点：有良好的程序局部性，缓存友好，对于内部排序，在实际应用中性能优于希尔排序和堆排序，因为i和j的移动是顺序的，而希尔排序，数组下标移动是按照间隔序列移动的，堆排序，数组下标移动按照2的幂移动，对于大数组，缓存容易丢失。

是不稳定的，因为i和j指向的元素等于主元时，会交换，改变了原始顺序。

对于小数组，实际性能不如插入排序，因此通常和插入排序联合使用，当子序列较长时，使用快速排序，当子序列长度小于阈值cutoff时，递归结束，使用插入排序。

```cpp
//median3算法选主元，将左、中、右的数字排序，将中位数移动到序列right-1，返回中位数
inline int Median3(int arr[],int left,int right){
    int mid=(left+right)/2;
    if(arr[left]>arr[mid])
        Swap(&arr[left], &arr[mid]);
    if(arr[mid]>arr[right]){
        Swap(&arr[mid],&arr[right]);
        if(arr[left]>arr[mid])
            Swap(&arr[left], &arr[mid]);
    }
    Swap(&arr[mid],&arr[right-1]);
    return arr[right-1];
    //处理[left,right-2]序列
}
//left和right表示序列的开始和结束位置,[left,right]
void DoQuickSortRec(int arr[],int left,int right){
    const static int cutoff=CUTOFF;
    if(right-left+1>cutoff){
        int pivot=Median3(arr,left,right);
        //i从左往右移动，直到第一个>=主元的位置，j从右往左移动，直到第一个<=主元的位置，交换i,j位置的值，循环移动i,j，直到i>=j,交换i和主元的值
        int i=left,j=right-2;
        while(1){
            while(arr[i]<pivot) ++i;
            while(arr[j]>pivot) --j;
            if(i>=j) break;
            Swap(&arr[i],&arr[j]);
            ++i;
            --j;
        }
        Swap(&arr[i],&arr[right-1]);
        //此时主元已经在正确的位置上，递归对主元左侧的序列和右侧的序列，执行快速排序
        DoQuickSortRec(arr, left, i-1);
        DoQuickSortRec(arr,i+1,right);
    }else{
        //序列长度小于cutoff值时，结束递归
    }
}
```

```cpp
//快速排序（非递归）
void Quick_sort(int arr[],int N){
    const static int cutoff=CUTOFF;
    int left=0,right=N-1,st[64],top=-1;
    while(top>-1||right-left+1>cutoff){
        //类似于二叉树的先序遍历，先循环访问较短的子序列，直到较短子序列的长度小于cutoff
        while(right-left+1>cutoff){
            int pivot=Median3(arr,left,right);
            //i从左往右移动，直到第一个>=主元的位置，j从右往左移动，直到第一个<=主元的位置，交换i,j位置的值，循环移动i,j，直到i>=j,交换i和主元的值
            int i=left,j=right-2;
            while(1){
                while(arr[i]<pivot) ++i;
                while(arr[j]>pivot) --j;
                if(i>=j) break;
                Swap(&arr[i],&arr[j]);
                ++i;
                --j;
            }
            Swap(&arr[i],&arr[right-1]);
            if(right-i>i-left){
                st[++top]=i+1;
                st[++top]=right;
                right=i-1;
            }else{
                st[++top]=left;
                st[++top]=i-1;
                left=i+1;
            }
        }
        //将left和right转向栈顶子序列,
        right=st[top--];
        left=st[top--];
    }
    Insertion_sort(arr,N);
}
```

## 表排序
如果待排序列中的元素占用空间较大，而排序的过程中，需要多次移动序列的元素，代价较大，可以将元素的地址（数组下标或指针）保存到一个数组中，对地址数组进行排序，然后按照地址数组的顺序读取原序列中的元素，比如指针数组、数据库索引。

<!-- 这是一张图片，ocr 内容为： -->
![](/assets/images/1776389639270-0c74d523-dd85-43d4-bdf0-38c03cb6ceaa.webp)

单射：定义域中，任意2个不同的x，对应到值域的y一定不同的映射，不重复。

满射：值域的任意一个y，都有一个x与它对应的映射，不遗漏。

双射：既是单射，又是满射的映射。

N个数字的排列（排列）：1~N的任意一种顺序的序列，序列元素不重复，刚好是1~N，假设集合S包含1~N，排列是S到自身的双射，比如3,5,2,1,7,0,4，对应的映射是0->3，1->5, 2->2, 3->1, 4->7, 5->0, 6->4。

N个数字的排列由若干个独立(互不相交)的环组成，证明：取排列中任意一个数字x1，对它不断的应用映射，得到N+1个数字，因为排列中的数字有限个，所以这N+1个数字x1,x2....xN+1一定存在xi=xj，对xi和xj执行逆函数，得到xi-1和xj-1，因为是双射，xi-1和xj-1一定相等，不断的应用逆函数，得到x1=xj-i+1，所以从任意数字x1出发，一定能回到自身形成环路。假设2个不同的环a,b相交，有公共数字ai=bi，对ai和bi不断的应用映射，得到的序列一定完全相同，因此这2个环路相同，和假设矛盾。

由地址数组的排序结果，对原序列进行排序：如下，地址数组是排列，对每一个环，保存环的起始元素，将环的下一个原序列元素复制到前一个位置，同时将地址数组更新为原序列的地址值，表示当前位置已经放对位置，复制之前，判断复制的目标位置是否已经放对位置，如果是，说明环路结束。取下一个table[i]!=i的位置，作为环的起始。

时间复杂度：最好情况下，已经有序，O(1)，因为复制到Temp前，会先判断table[i]==i。最坏情况，每2个元素形成环路，N/2个环，每个环复制3次，O(3N/2)，平均是O(mN)，m是每个元素的复制时间。

```cpp
//表排序,返回排序后的索引数组
int* Table_sort(book arr[],int N){
    //移动结构数组的成本较高，使用索引数组，移动索引,初始化索引数组为0~N-1
    int *table=malloc(sizeof(int)*N);
    for(int i=0;i<N;i++){
        table[i]=i;
    }
    int temp,j;
    for(int i=1;i<N;i++){
        temp=table[i];
        for(j=i-1;j>=0&&CompareBook(&arr[temp],&arr[table[j]])<0;j--){
            table[j+1]=table[j];
        }
        table[j+1]=temp;
    }
    return table;
}
//根据排序后的索引数组，对原序列进行排序,移动次数是O(3N/2)
void Table_sort_physical(book bs[],int table[],int N){
    //table是一个排列，由若干个互不相交的环组成
    book temp;
    int j,nextJ;
    for(int i=0;i<N;i++){
        j=i;
        nextJ=table[j];
        if(nextJ!=j){
            temp=bs[j];
            while(nextJ!=table[nextJ]) {
                bs[j]=bs[nextJ];
                table[j]=j;
                j=nextJ;
                nextJ=table[j];
            }
            bs[j]=temp;
        }
    }
}
```

## 基数排序
冒泡排序、选择排序、希尔排序、堆排序、归并排序、快速排序是基于比较的排序，桶排序和基数排序不是基于比较的排序。

桶排序：如果待排序列元素的取值范围较小，可以创建一个桶数组，桶的数组地址代表待排序列元素的值，遍历待排序列，根据元素的值将它保存到对应地址的桶，遍历桶数组，就可以得到有序序列，桶可以用链表实现，也可以用counting sort。时间复杂度是O(M+N)，因为从桶数组取值的时候，需要遍历每个数组元素，获取其中不为空的桶的元素，空间复杂度是O(M+N)，N是待排序列的元素个数，M是取值范围。

![](/assets/images/1776402910662-57f32134-77a6-4c4e-8f1b-644059c81059.webp)

基数排序：用于解决桶排序M>>N时，时间复杂度不是线性的，大部分桶是空的，空间利用率低的问题，基数是序列元素每一位的取值个数，等于进制数，比如10进制，基数是10，2进制的基数是2，桶的个数等于基数，分为次位优先（LSD,Least Significant Digit）和高位优先（MSD，Most Significant Digit）。

LSD：如下，从最低位开始，根据每个元素某一位的值，将元素放到对应的桶中，外层循环轮数等于元素位数的最大值，每一轮需要按顺序遍历当前桶中的元素，从而同一个桶中的元素，当前位相同，次位有序。是稳定的，因为每轮是按照原始序列的顺序读取的，相等的元素，按照原始序列的顺序放入桶中。时间复杂度是O(P*(N+B))，P是最高位数，N是待排序列的长度，B是基数，每一轮需要遍历每个桶，不管是否有元素。

LSD的每一轮排序，必须稳定，否则最终得不到有序序列，因为每一轮只排序其中一位，必须保持之前排序的低位的顺序。

Counting Sort：如下，使用数组来保存桶的元素，具有更好的程序局部性，缓存友好。

```cpp
使用3个数组：
A[0..N-1]       原数组
count[0..B-1]   计数数组
output[0..N-1]  输出数组

digit获取当前位，count[i]保存了第i个桶的元素个数。
for i = 0..N-1
    d = digit(A[i])
    count[d]++

计算前缀和，count[i]表示第0~i个桶的元素个数，也就是第i个桶在数组中的结束位置。
for i=1..B-1
    count[i] += count[i-1]

count[i]是第i个桶在数组中的结束位置，所以从后往前放置桶，因此读取A的顺序也需要从后往前，从而确保稳定性，也就是同一个桶的元素，顺序必须和A中的顺序一致。
for i=N-1..0
    d = digit(A[i])
    pos = count[d] - 1
    output[pos] = A[i]
    count[d]--
```

MSD：从最高位开始，对序列元素进行分桶，之后桶之间的元素不需要再比较，对桶内的元素，递归对次高位分桶，直到最低位。分桶也可以使用Counting Sort，时间复杂度？。

区别：LSD不管低位是否相同，需要比较每一位的值，MSD高位不同时，不需要比较低位，LSD适用于位数较为固定的元素序列，MSD适用于前缀差异大，对位数没有要求，比如字符串，MSD使用场景：Spark分区、ES倒排索引、基数树、路由表(IP地址)、URL、文件路径，LSD使用场景：整数ID。

```cpp
//获取num二进制的位数，使用二分法的思想获取后缀0的个数，再加1
int GetBitNum(int num){
    int suffixN=0;
    if(num>>16!=0) suffixN+=16;
    if(num>>(suffixN+8)!=0) suffixN+=8;
    if(num>>(suffixN+4)!=0) suffixN+=4;
    if(num>>(suffixN+2)!=0) suffixN+=2;
    if(num>>(suffixN+1)!=0) suffixN+=1;
    return suffixN+1;
}
//根据arr中元素的第nth个8位数的值，分桶
void MoveToBucket(int arr[],int bucket[],int count[],int N,int nth){
    const int radixNum=RADIX_NUM,radixNumMask=RADIX_NUM_MASK,radixBitNum=RADIX_NUM_BITNUM;
    nth*=radixBitNum;
    memset(count,0,sizeof(int)*radixNum);
    for(int i=0;i<N;i++){
        count[arr[i]>>nth&radixNumMask]++;
    }
    //计算前缀和
    for(int i=1;i<radixNum;i++){
        count[i]+=count[i-1];
    }
    //从后往前遍历，确保排序的稳定性
    int bucketN;
    for(int i=N-1;i>=0;i--){
        //arr[i]对应第几个桶
        bucketN=arr[i]>>nth&radixNumMask;
        bucket[--count[bucketN]]=arr[i];
    }
}
//基数排序，次位优先（LSD，Least Significant Digit）,用于范围确定，位数较为固定的数的序列，这里实现1~N范围的N个正整数序列
void Radix_sort(int arr[],int N){
    const int radixShift=RADIX_NUM_SHIFT, radixNum=RADIX_NUM;
    int *bucket=malloc(sizeof(int)*N);
    int loopN=(GetBitNum(N)>>radixShift)+1;//比如17位数，0x20000,4位数，0x1f
    //使用Counting Sort分桶，每8位一个桶
    int count[radixNum];
    //将arr分成每8位一轮循环，每轮处理第i个8位数，桶的个数是256，表示第i个8位数的取值（0~255）
    for(int i=0;i<loopN;i++){
        MoveToBucket(arr,bucket,count,N,i);
        // Print1D(bucket,N);
        i++;
        //低2位是0：0000=0,0100=4,1000=8,1100=c, 低2位是1：0001=1,0101=5,1001=9,1101=d，低2位是2:0010=2,0110=6,1010=a,1110=e, 低2位是3：0011=3,7,b,f    高2位是0：0000=0,0001=1,0010=2,0011=3, 高2位是1：4,5,6,7, 高2位是2: 8,9,a,b, 高2位是3：c,d,e,f
        MoveToBucket(bucket,arr,count,N,i);
        // Print1D(arr,N);
    }
    free(bucket);
}
```
